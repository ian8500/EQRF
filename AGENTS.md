# Repository Working Instructions

- Review `README.md` during every task.
- Update `README.md` whenever behaviour, setup, deployment, dependencies, data model, Admin workflow, public UI, routes, or tests change.
- If no README change is needed, state this in the final response.
- Keep `README.md` factual and aligned with the code.
- Do not document planned features as if they already exist.
- Run tests before committing where possible.
- Prefer small, focused commits.
- Do not remove operational data, PDFs, audit logs, or checklist content unless explicitly requested.
